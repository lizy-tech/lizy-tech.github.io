java并发--概述

发表于 2018-10-08 | 分类于 java ， 并发
字数统计: 328 字 | 阅读时长 ≈ 1 分钟

概述

并发编程相比 Java 中其他知识点的学习门槛较高，从而导致很多人望而却步。但无论是职场面试，还是高并发/ 高流量系统的实现，却都离不开并发编程，于是能够真正掌握并发编程的人成为了市场迫切需求的人才。
笔者最近突然想好好整理一下java的并发知识。主要是介绍jdk1.5之后出现的JUC（java.util.concurrent包），当然也会介绍一些线程的基本问题。

JUC整体结构

JDK 1.5之前，我们必须自己编写代码实现一些并发编程的逻辑；之后到了JDK 1.5，Doug Lea解救了广大Java用户，在JDK里特意设计并实现了一套JUC的框架，给大家提供了非常好的并发编程体验。
JUC的整体结构如下：

atomic

juc包下的原子类，主要是一些原子操作的实现：

locks

主要实现一些并发编程中会用到的锁

collections

容器在我们编程的过程中是比较常用的，JUC下提供了一些并发安全的容器，主要如下：

exector

这一部分主要是与线程池相关的类

tools

这一部分主要是JUC的工具类
